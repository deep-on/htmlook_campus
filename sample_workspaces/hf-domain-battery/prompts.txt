LLM-agnostic prompts for the domain-battery scenario — five English
prompts. Drop any of these into your terminal of choice with the
HTMLook MCP attached:

    claude --continue
    ollama run llama3.2
    codex
    gpt-5  (or whatever shell you wire up)

The MCP moves (htmlook_pane_pair, htmlook_active_file,
htmlook_apply_edit, htmlook_cite) are identical regardless of which
LLM is driving. That is the whole point of this scenario: the LLM
brand is interchangeable, the workflow is not.

Per the project language-consistency rule this file is English-only.
Korean variants live alongside as separate files when needed.

────────────────────────────────────────────────────────────────────
[1] Simple request (main)
────────────────────────────────────────────────────────────────────

Build a 15-second teaser called "Battery Bench 2026" — a kinetic
benchmark card for an EV-battery competitive report. Deep-navy
canvas (#0b1a2e), lime accent (#a3e635), amber highlight (#fbbf24).
Four scenes:

  Scene 1 (0-3s)   intro tag "BATTERY BENCH 2026" + a short lime
                   rule that swipes in left-to-right.
  Scene 2 (3-7s)   kinetic typography on the three benchmark
                   numbers stacked: "320 Wh/kg", "10C peak",
                   "8000 cycles". Digit-cluster stagger.
  Scene 3 (7-11s)  the same numbers settle into a small bar chart
                   row behind three vendor silhouettes (monochrome
                   only — Vendor A / Vendor B / Vendor C, no real
                   logos).
  Scene 4 (11-15s) wordmark close — "Battery Bench 2026" with a
                   back.out ease, URL "batterybench.io" types in
                   below in mono.

Lint clean, then render. Background gradient drifts gently the full
15 s.

[2] Refinement — slow the lead figure
────────────────────────────────────────────────────────────────────

Look at the right pane around 0:04 — "320 Wh/kg" snaps in too
fast. Slow the digit-cluster stagger from 0.04 → 0.08, and hold
the figure on screen for 0.6 s longer before the unit underline
sweeps in. Same 4 s scene window.

────────────────────────────────────────────────────────────────────
[3] Refinement — vendor silhouettes too dim
────────────────────────────────────────────────────────────────────

The three vendor silhouettes in scene 3 are barely visible against
the navy. Lift their fill to rgba(220, 234, 252, 0.42) and add a
1 px lime underline beneath each silhouette as it lands. Stagger
the three by 200 ms so the eye reads them left-to-right.

────────────────────────────────────────────────────────────────────
[4] Recompute one row from a cited PDF region
────────────────────────────────────────────────────────────────────

Use htmlook_cite on the right-pane PDF region "vendor_datasheet.pdf
page 4, Table 2, row 'cycle life @ 1C'" — the spec there says
8 200 cycles, not the 8 000 we have in scene 2. Update the third
benchmark to "8200 cycles" and re-render. Note the source page in
a // comment above the literal so we keep the citation auditable.

Re-author this composition for 1080×1920 portrait so it fits the
