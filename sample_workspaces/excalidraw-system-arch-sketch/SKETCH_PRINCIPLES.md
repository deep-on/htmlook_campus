# 5분 Sketch 원칙

빠른 화이트보드 sketch — formal 다이어그램 (D2/Mermaid) 만들기 전 단계.

## Do
- 5분 안에 그릴 수 있어야 함 (그렇지 않으면 너무 detailed)
- 박스 + 화살표만 — UML/BPMN 안 씀
- 한 페이지에 끝남
- 다음 단계 (formal) 의 input

## Don't
- 색깔 너무 많이 — 강조 1-2 색만
- icon 라이브러리 — 손그림이 의도
- 레이어 / 그룹 — 평면

## 예시 collection
- `diagrams/api-quick.excalidraw` — REST API + DB + cache
- `diagrams/data-pipeline.excalidraw` — ETL flow
- `diagrams/event-flow.excalidraw` — Kafka producer/consumer
- `diagrams/auth-flow.excalidraw` — OAuth dance
