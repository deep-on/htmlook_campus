# 튜토리얼

> 앱 안에서 진행되는 가이드 워크스루. **Help → Tutorials** 로 선택.

## 튜토리얼이란

튜토리얼은 *HTMLook 안에서* 진행되는 단계별 레시피 — 블로그로 탭 전환 안 함, 스크린샷 복붙 안 함. 각 단계가 클릭할 정확한 버튼/영역을 강조하고, (선택적으로) 연습용 샘플 파일을 로드하고, (AI 가 관여하면) 보낼 준비된 프롬프트를 어시스턴트에 떨궈줍니다.

일시정지 · 세션 간 재개 · 건너뛰기 · 중단 가능. 완료 ✅, 미완 ▶.

## 위치

- **Help 메뉴 → Tutorials** — 전체 라이브러리
- **첫 실행 onboarding** — HTMLook 이 "주로 어떤 일을 하시나요?" 묻고 그 역할에 맞는 튜토리얼 제안
- **사이드바 (샘플 워크스페이스 열려 있을 때)** — 그 샘플에 맞는 튜토리얼이 상단에 제안

## 라이브러리 (빌트인)

13 페르소나의 구체 워크플로우. 각 unit 5분 정도.

| # | 튜토리얼 | 할 일 |
|---|---|---|
| 01 | 회의 transcript 에서 마크다운 노트 | transcript 파일 drop, 어시스턴트에게 화자별 정리 부탁, PDF export |
| 02 | 한 페이지 PDF 리포트 다듬기 | 주석, 인용 추가, 레이아웃 재생성 |
| 03 | 짧은 영상 스토리보드 | 영상 import, 씬 마크, 챕터 제목 드롭 |
| 04 | 계약서 조항에 코멘트 | PDF 열기, 조항 하이라이트, Paint 스케치 첨부, 레드라인 제안 요청 |
| 05 | 논문 요약 | PDF 열기, outline + 3-bullet abstract 요청, Markdown 저장 |
| 06 | 노트에서 슬라이드 덱 | 마크다운 열기, Heading 1 을 슬라이드 제목으로, `.pptx` export |
| 07 | 음성 메모 → action item | 메모 녹음 → transcribe → 어시스턴트가 action item 추출 |
| 08 | 두 리포트 초안을 듀얼 뷰에서 비교 | 둘 열기, ⌘⇧D, sync scroll, Paint 로 차이 표시 |
| 09 | UI 버그 스크린샷 주석 | 이미지 drop, Paint 로 동그라미, 예상 fix 요청 |
| 10 | 문서 섹션 번역 | 텍스트 선택, 어시스턴트에게 번역 요청, 인라인으로 paste |
| 11 | 손그림 스케치로 Mermaid 다이어그램 | 스케치 이미지 drop, 어시스턴트에게 Mermaid 변환 요청 |
| 12 | 워크스페이스를 공유 번들로 | 파일 선택, Export → HTML self-contained, 이메일 첨부 |
| 13 | 자주 쓰는 프롬프트를 재사용 버튼으로 | 터미널에 프롬프트 타이핑, 책갈피 아이콘으로 preset 저장 |

## 각 튜토리얼의 구성

모든 단계가 최대 4 가지 ingredient:

1. **클릭할 것** — 타겟 버튼/메뉴가 부드러운 펄스 + 화살표로 강조
2. **샘플 파일** *(선택)* — *Load sample* 버튼이 검증된 파일을 워크스페이스의 `__wizard__/` 하위에 떨어뜨림 — 본인 파일 찾지 않아도 됨
3. **준비된 프롬프트** *(선택)* — 원클릭 버튼이 미리 작성된 프롬프트를 활성 터미널 또는 AI 어시스턴트에 paste
4. **Next / Skip / Stop** 하단

## 샘플 워크스페이스

일부 튜토리얼이 형제 폴더로 작은 샘플 워크스페이스 설치 — 레슨과 실제 파일이 섞이지 않도록. 샘플 폴더는 `htmlook-sample-<slug>/` 접두사. 튜토리얼 후 삭제 가능.

## 재개 + 리셋

진행이 자동 저장. 앱 닫고 다시 열면 중단 지점에서 재개 제안. 새로 시작하고 싶으면 라이브러리의 *Reset* 버튼이 그 튜토리얼의 진행 wipe.
